What is Agentic AI?

Agentic AI refers to AI systems that can independently perceive their environment, make decisions, take actions, and learn from outcomes — with minimal human guidance.

Key Idea: An AI agent doesn't just answer questions — it does things. It reasons about what to do, uses tools, and works toward goals on its own.

  • Goes beyond simple chatbots and autocomplete
  • Can break complex tasks into steps and execute them
  • Interacts with the real world through tools and APIs

Traditional AI vs. Agentic AI

Traditional AI / Chatbots

  • Responds only when prompted
  • Single turn: question in, answer out
  • No memory across interactions
  • Cannot take actions in the world
  • Human does all the planning

Agentic AI

  • Proactively works toward goals
  • Multi-step reasoning and execution
  • Maintains memory and context
  • Uses tools (search, code, APIs)
  • Plans, acts, and self-corrects

LLMs: The Brain of the Agent

Large Language Models (LLMs) like Claude, GPT, and Gemini serve as the reasoning engine inside modern AI agents.

  • Natural language understanding — interpret complex user requests
  • Planning — break big tasks into smaller steps
  • Decision making — choose which tool to use and when
  • Code generation — write and debug code on the fly
  • Self-reflection — evaluate their own outputs and retry if needed

The LLM doesn't just generate text — it orchestrates an entire workflow of reasoning, tool use, and evaluation.

The ReAct Pattern (Reasoning + Acting)

ReAct is a foundational pattern where the agent alternates between thinking and doing.

Thought
"I need to find..."
→
Action
search("query")
→
Observation
Results returned
→
Thought
"Now I know..."
  • The agent thinks out loud about what it needs to do
  • It then takes an action (e.g., call a tool)
  • It observes the result and decides the next step
  • This loop repeats until the task is complete

Tool Use: How Agents Interact with the World

Agents become truly useful when they can use tools — external functions that extend their capabilities beyond text generation.

  • Web Search — look up current information online
  • Code Execution — write and run Python, JavaScript, etc.
  • File Operations — read, write, and manage files
  • API Calls — interact with databases, services, and platforms
  • Calculators & Data Tools — perform precise computations

Example: "What's the weather in Nairobi?" — The agent calls a weather API, gets live data, and presents the answer.

Agent Memory: Remembering and Retrieving

Without memory, agents forget everything between interactions. Memory systems give agents continuity and context.

Short-Term Memory

  • Current conversation context
  • Working notes during a task
  • Limited by context window size

Long-Term Memory

  • Stored in vector databases
  • Persists across sessions
  • Retrieved via similarity search (RAG)

RAG (Retrieval-Augmented Generation): The agent searches a knowledge base for relevant info and includes it in its reasoning.

How Agents Plan

For complex tasks, agents need to plan ahead rather than just react step-by-step.

  • Plan-and-Execute: First create a full plan, then execute each step
  • Chain-of-Thought: Reason through the problem step-by-step before acting
  • Tree-of-Thought: Explore multiple reasoning paths and pick the best one
  • Reflexion: After completing a task, reflect on what went wrong and improve

Example — Plan-and-Execute:
1. Research the topic → 2. Outline the article → 3. Write each section → 4. Review and edit → 5. Format and deliver

Safety, Ethics & Guardrails

Autonomous agents are powerful — but they need boundaries to operate safely.

  • Hallucination Risk: Agents can act on incorrect information — always verify critical outputs
  • Infinite Loops: Without limits, agents may repeat actions endlessly — set max iterations
  • Harmful Actions: Agents with file/system access need sandboxing and permission controls
  • Human-in-the-Loop: For high-stakes decisions, require human approval before execution
  • Transparency: Log all agent reasoning and actions for auditability
  • Bias & Fairness: Agents inherit biases from their training data — test for and mitigate bias
